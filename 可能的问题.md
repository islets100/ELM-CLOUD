# ELM-CLOUD 

按顺序照做即可。

## 1. 准备

- JDK 8
- Maven
- Docker Desktop（MySQL和Redis在docker上跑，方便统一版本）
- IDEA

## 2. 拉代码后先做这两步

在项目根目录（`ELM-CLOUD`）执行：

```powershell
Copy-Item .env.example .env
```

然后打开刚刚这个文件.env确认端口：

- 默认是 `MYSQL_PORT=3306`、`DB_PORT=3306`

如果你电脑本机已经装了 MySQL 并占用了 3306，而你又不想停掉它，可以换一个端口：

```env
MYSQL_PORT=3307
DB_PORT=3307
```

## 3. 启动数据库和缓存（Docker）

```powershell
docker compose up -d
docker compose ps
```

看到 `elm-mysql` 和 `elm-redis` 都是 `Up` 即可。

## 4. 启动后端服务

先启动注册与配置中心，再启动业务服务。

### 4.1 先启动 eureka

```powershell
cd elm-eureka-server
mvn spring-boot:run
```
也可以使用IDEA启动

### 4.2 再启动 config-server

配置中心路径（固定路径），下面的路径是我本机的路径，需要改成你自己电脑的绝对路径，使用绝对路径为了方便不同文件夹都能访问到config：

- [`elm-config-server/src/main/resources/application.yml`] 第12行配置
    file:///e:/CourseData/software_advanced_practice/ELM-CLOUD/elm-configs（这是我的）
- 其中 `search-locations` 要改成你自己电脑上的 `elm-configs` 绝对路径（`file:///.../elm-configs`）

然后启动：

```powershell
cd ../elm-config-server
mvn spring-boot:run
```

### 4.3 启动业务服务（每个服务一个终端）

```powershell
cd ../elm-user-server; mvn spring-boot:run
cd ../elm-business-server; mvn spring-boot:run
cd ../elm-food-server; mvn spring-boot:run
cd ../elm-cart-server; mvn spring-boot:run
cd ../elm-delivery-server; mvn spring-boot:run
cd ../elm-wallet-server; mvn spring-boot:run
cd ../elm-order-server; mvn spring-boot:run
cd ../elm-gateway-server; mvn spring-boot:run
```

`elm-point-server` 默认含 RabbitMQ 配置，先不启动它也可以。

## 5. 启动成功怎么判断

- Eureka 页面能打开：`http://localhost:13000/`
- Gateway 端口启动成功（日志显示 14000）  
- 业务服务启动日志里没有连续报错（尤其是数据库连接和 eureka 心跳）

## 6. 最常见报错与处理

## 报错 A：`Connection refused: 127.0.0.1:13000`

原因：Eureka 没启动。  
处理：先启动 `elm-eureka-server`，再启动其他服务。

## 报错 B：MySQL 起不来 / 3306 被占用

处理：

1. 把 `.env` 的 `MYSQL_PORT`、`DB_PORT` 都改成 `3307`
2. 重新启动 Docker：

```powershell
docker compose down
docker compose up -d
```

## 报错 C：`Table ... doesn't exist`

原因：只创建了数据库，没有导入业务表。  

## 报错 D：config-server 启动后读不到配置

原因：`search-locations` 路径不是你本机真实路径。  
处理：改成你本机 `elm-configs` 的绝对路径（`file:///` 开头）。
